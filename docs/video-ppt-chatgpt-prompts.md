# 视频 PPT 生成提示词

这份文档把内容拆成多个独立对话提示词，适合你在 ChatGPT 里一条一条发送。这样比一次性生成整套 PPT 更稳，也更容易保持风格一致。

## 通用约束

每次发新提示词时，都先把这一段保留在对话开头：

```text
请保持和前一页一致的视觉风格，不要换主题，不要换字体气质，不要换配色。页面适合 16:9 视频录制，文字少一点，重点清楚一点，适合直接播放。文案要像真人在讲，不要像模板，不要有 AI 感。
```

## 对话 1：先定风格

```text
请先不要生成 PPT 内容，先帮我定一套统一的 16:9 演示风格，用于介绍我的小项目和第三 Tab demo。

要求：
1. 整体风格统一，适合视频录制。
2. 偏克制、清晰、轻微科技感，不要花哨。
3. 适合中文讲解，页面不要太密。
4. 请给我 1 套主风格方案，并说明配色、字体气质、版式、图标风格和页面留白。
5. 只输出风格方案，不要开始写正文页。
```

## 对话 2：封面页

```text
请基于上一轮确定的风格，生成第 1 页封面。

这一页的内容是介绍我的小项目 TRPG Lucius Helper，重点是“一个面向 TRPG Keeper 的轻量工具集”。

要求：
1. 只生成 1 页。
2. 标题短一点，适合视频开场。
3. 页面上只保留一个核心句子和少量辅助信息。
4. 给我这一页的口播文案，语气自然一点，不要像念稿。
5. 给我这一页的画面布局建议。
```

## 对话 3：问题背景页

```text
请继续生成第 2 页，只做一页。

这一页要讲清楚我为什么做这个项目，也就是我想解决什么问题。

请围绕这三个点来写：
1. 信息分散，不方便回看。
2. 临场编辑成本高，流程不够直观。
3. 需要一个能展示、能修改、能导出的统一界面。

要求：
1. 文字尽量短，像演示页，不像说明书。
2. 给我一段自然的口播文案。
3. 给我一个适合视频播放的布局建议。
```

## 对话 4：第三 Tab Demo 页

```text
请继续生成第 3 页，只做一页。

这一页要介绍第三 Tab 的 demo，重点展示它是一个音效键盘的自定义播放TAB。

请表达清楚这几个点：
1. 内置默认跑团音效。
2. 完全支持自定义音效和标题。
3. 支持同时叠加播放多个音效。
4. 支持自定义某一个音效的音量倍率。
5. 我的思考：现在AI vibeCoding那么多，跑团工具如雨后春笋。但我认为面团最重要的还是KP控场，PL之间面对面而非面对屏幕。因此我的工具并非完备的什么车卡工具，我希望把这些内容留在线下，只是KP的中控板可以更智能。

要求：
1. 这一页要适合衔接一段我录制的视频demo。
2. 页面正文不要超过 3 到 4 条短句。
3. 口播要像我在边看边讲，语气轻松自然。
4. 给我直接生成一份PPT图片。
```

## 对话 5：核心亮点页

```text
请继续生成第 4 页，只做一页。

这一页要讲这个 demo 的核心亮点，重点是：
1. 快速编辑。
2. 关系联动。
3. 导入导出和复用。

要求：
1. 每个亮点只用一句短文案。
2. 页面要像产品展示，不像技术汇报。
3. 给我一段更像真人说话的口播文案，不要太正式。
4. 给我页面布局建议，最好是卡片式或者三栏式。
```

## 对话 6：实现方式页

```text
请继续生成第 5 页，只做一页。

这一页要讲实现方式，但要讲得很轻，不要太技术化。重点是：
1. React 负责界面组织。
2. 状态和数据流尽量本地化管理。
3. 流程图、布局和导入导出都在前端完成。

要求：
1. 只保留最必要的技术信息。
2. 口播要像项目作者自己在讲，不要像答辩。
3. 页面上可以出现少量技术词，但不要堆太多。
4. 给我一个适合录屏讲解的布局建议。
```

## 对话 7：收尾页

```text
请继续生成第 5 页，只做一页。

这一页是结尾，要自然收住，表达这个 demo 已经能展示一个完整的小工作流，后续还会继续扩展。

要求：
1. 结尾要干净利落。
2. 语气要像真人收尾，不要有“感谢观看”那种太模板的感觉。
3. 给我一段很短的收尾口播。
4. 给我一句适合放在页面上的收束文案。
```

## 视频开场文案

```text
这是我做的一个小项目，名字叫 TRPG Lucius Helper。它想做的事情很简单，就是把 TRPG 现场常用的任务、线索和流程操作放到一个页面里，尽量少切换、少折腾。下面我用第三 Tab 的一个 demo，简单看一下它现在做到什么程度。
```

```text
前面这一页主要是在说问题和目标，接下来我直接看 demo 本身。你会更容易看出它不是单纯展示内容，而是把这些内容做成了能编辑、能联动的工作界面。
```

## 备用短标题

如果你想让页面标题更像视频字幕，可以直接用这些短标题：

```text
1. 一个小项目
2. 我为什么做它
3. 第三 Tab Demo
4. 核心亮点
5. 简单实现
6. 阶段性收尾
```

## 剪映常用小技巧

如果这类口播最后要用剪映来收尾，我一般会先记住这几件最实用的小操作：

```text
1. 先用“分割”把废话、停顿和重复句切掉，口播会立刻利索很多。
2. 先开“自动识别字幕”或“智能剪口播”，把粗剪和字幕打底，后面只做微调。
3. 用“关键帧”做一点点推近、拉远或位置变化，画面不容易太平。
4. 口播音频先做“降噪”和“响度统一”，再压背景音乐，整体会更稳。
5. 需要强调某一段时，用变速、定格、放大或者画中画，往往比堆特效更干净。
6. 素材一多就把主画面、补充画面和字幕分开处理，多轨时间线会省很多回改时间。
7. 导出前最后看一遍封面、字幕边距和节奏，这三处最容易影响成片质感。
```

## 剪映收尾口播

```text
如果这类内容后面是用剪映来剪，我最常用的其实就这几步：先粗剪，再补字幕，再处理音频，最后只留少量关键帧和必要的转场。工具不用堆太多，把节奏理顺，成片就会干净很多。
```
