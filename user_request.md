# TRPGLuciusHelper - 用户需求文档

## 项目概述

TRPGLuciusHelper 是一个 TRPG（桌面角色扮演游戏）辅助工具项目，以 Web 形式运行，支持本地部署或服务器部署。主要用于支持网络和线下跑团的助手功能。

## 核心需求

### 1. 基础要求

- Web 端提供一个简洁的配置页面，包含多个附属功能
- 需要适配 macOS、Windows、Android、iOS 多平台
- 技术选型偏向前沿技术栈
- 注重文档维护（包括技术文档和 AI 工作记录文档），保证人工随时可以介入

### 2. 核心功能 - 三个 TAB

#### TAB1 - 个人介绍与模组链接

- 展示和维护个人介绍信息
- 管理不同状态的模组：已备过的、待准备的、已经带过的
- 更新带人跑团的免责声明和要求
- 支持随时更新维护

#### TAB2 - 模组工具页面

- 点击具体模组或上传自定义模组，显示可视化的拖拽线索工具
- 支持标注线索之间的关联关系
- 点击线索可记录当前模组中哪些关键线索已被取得（置灰处理）
- 使用有向无环图（DAG）实现线索网络的拖拽与渲染
- 需要较好的图形渲染效果（考虑类似 WebGL 级别的图形技术）

#### TAB3 - 个人博客与杂谈

- 文章更新与展示
- 支持博客和杂谈等多种内容类型

### 3. 数据导入导出

- TAB1 和 TAB2 均支持数据的导入和导出
- 使用 JSON 格式
- 便于在不同设备的 localhost 之间迁移数据

### 4. TAB2 线索 DAG 细化需求（2026-04-17 用户反馈）

#### 4.1 多对多关系
- 线索/物品之间是**多对多**关系：一个线索可以引出多个线索，一个线索也可以被多个线索引用
- 示例模组需要展示出这种复杂的多对多图结构，而非简单的树形

#### 4.2 边触发发现机制
- **核心交互改为点击"边"（关系）**，而非点击节点
- 例如：线索A --阅读--> 线索B，点击"阅读"这条关系后，A 和 B 都标记为已发现
- 直接点击节点本身标记发现属于**特殊情况**，点击后需要弹窗补充说明（为什么这个线索被直接发现）

#### 4.3 一键排版
- 支持一键自动排版功能
- 不改变线索的发现状态
- 将所有节点按从上到下的层次布局重新排列

### 5. 美术风格需求（2026-04-17 用户反馈）

- 整体美术风格偏向**克苏鲁/宇宙恐怖**风格
- 暗色主题为主
- 需要专业的 UI 设计调研，不能太丑
- 配色、字体、动效需统一风格
