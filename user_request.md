# TRPGLuciusHelper - 用户需求文档

## 项目概述

TRPGLuciusHelper 是一个 TRPG（桌面角色扮演游戏）辅助工具项目，以 Web 形式运行，支持本地部署或服务器部署。主要用于支持网络和线下跑团的助手功能。

## 核心需求

### 1. 基础要求

- Web 端提供一个简洁的配置页面，包含多个附属功能
- 需要适配 macOS、Windows、Android、iOS 多平台
- 技术选型偏向前沿技术栈
- 注重文档维护（包括技术文档和 AI 工作记录文档），保证人工随时可以介入

### 2. 核心功能 - 三个 TAB

#### TAB1 - 个人介绍与模组链接

- 展示和维护个人介绍信息
- 管理不同状态的模组：已备过的、待准备的、已经带过的
- 更新带人跑团的免责声明和要求
- 支持随时更新维护

#### TAB2 - 模组工具页面

- 点击具体模组或上传自定义模组，显示可视化的拖拽线索工具
- 支持标注线索之间的关联关系
- 点击线索可记录当前模组中哪些关键线索已被取得（置灰处理）
- 使用有向无环图（DAG）实现线索网络的拖拽与渲染
- 需要较好的图形渲染效果（考虑类似 WebGL 级别的图形技术）

#### TAB3 - 个人博客与杂谈

- 文章更新与展示
- 支持博客和杂谈等多种内容类型

### 3. 数据导入导出

- TAB1 和 TAB2 均支持数据的导入和导出
- 使用 JSON 格式
- 便于在不同设备的 localhost 之间迁移数据

### 4. TAB2 线索 DAG 细化需求（2026-04-17 用户反馈）

#### 4.1 多对多关系
- 线索/物品之间是**多对多**关系：一个线索可以引出多个线索，一个线索也可以被多个线索引用
- 示例模组需要展示出这种复杂的多对多图结构，而非简单的树形

#### 4.2 边触发发现机制
- **核心交互改为点击"边"（关系）**，而非点击节点
- 例如：线索A --阅读--> 线索B，点击"阅读"这条关系后，A 和 B 都标记为已发现
- 直接点击节点本身标记发现属于**特殊情况**，点击后需要弹窗补充说明（为什么这个线索被直接发现）

#### 4.3 一键排版
- 支持一键自动排版功能
- 不改变线索的发现状态
- 将所有节点按从上到下的层次布局重新排列

### 5. 美术风格需求（2026-04-17 用户反馈）

- 整体美术风格偏向**克苏鲁/宇宙恐怖**风格
- 暗色主题为主
- 需要专业的 UI 设计调研，不能太丑
- 配色、字体、动效需统一风格

### 6. TAB2 自定义模组线索图创建（2026-04-19 用户需求）

#### 6.1 创建自定义模组线索图
- 支持创建自定义模组线索图，完成后保存导出到本地
- 初始线索永远是【模组开始】

#### 6.2 添加新线索
- 点击线索/物品框周围增加新线索，直接生成此线索到新线索的联系
- 联系有默认名称，也可以后续编辑
- 新线索框通过对话形式，点击确定才真正生成

#### 6.3 拖拽连线
- 长按线索/物品框边缘节点拖拽，拖到另一个线索/物品框生成单向联系

#### 6.4 二次编辑
- 支持二次编辑现有的线索/物品

#### 6.5 图片支持
- 支持导入图片到节点
- 展示模式下，如果节点有图片，支持点击放大图片查看

### 7. 新TAB3 音效键盘（2026-04-19 用户需求）

> 新Tab放在第三个位置，原来的TAB3博客顺次到第四个

#### 7.1 3D键盘UI
- 使用合适的技术实现一个模拟键盘3D的UI

#### 7.2 键盘监听
- 监听用户键盘输入：26个字母、空格、回车
- 检测到输入，对应的按钮也按下

#### 7.3 音效播放
- 支持播放音频小音效，类似于喜剧控场的效果
- 每一个按钮按下都有对应的音频播放
- 回车键特殊：点击后停止中断当前所有音频播放

#### 7.4 自定义音效
- 每一个按钮的音效都支持自定义上传
- 支持用JSON文件映射+音频文件文件夹的形式导入和导出

#### 7.5 默认音效
- 提供默认的初始音效和映射，便于实验使用

#### 7.6 视觉区分
- 未绑定音效的按钮可以有点击动效，但颜色需要置灰
- 和有音频的按钮区分开

## 8. TAB2 新增世界 Wiki（2026-05-27 用户需求）

- 在第二个 TAB 的工具浮窗中新增第二个功能项：`世界 wiki`
- UI 参考 Wikipedia 的信息架构，并结合金斯波特补充资料页面的氛围感，保持与现有站点风格统一
- Wiki 首页为人物 / 地点 / 事件 / 模组四类信息的检索页
- 页面右上角提供 `[当前PL：xx]` 按钮，逻辑与 BlogTab 的 PL 名称设置一致
- 初始需要提供金斯波特相关示例词条，包含至少 3 名人物、若干地点、若干事件与 1 个模组入口
- 不同词条之间支持超链接跳转
- Wiki 数据必须来自静态文件，后续 push 词条内容时不需要重新 build 整站
- 需要预留“黑色框遮盖文本”的隐藏档案机制：根据当前 PL 名称决定是否解锁；未解锁时点击提示“请探索更多故事解锁~”
