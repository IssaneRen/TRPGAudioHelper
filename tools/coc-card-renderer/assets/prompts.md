# ChatGPT Image 素材提示词

目标：生成可拼接到 CoC 人物卡图片上的统一素材。建议输出 PNG；装饰素材使用透明背景。

## 1. 人物卡固定结构空壳

```text
从零生成一张全新的竖版 1200x1800px CoC7th 调查员人物卡固定结构空壳，用作 Python 脚本后续填字的底图。

重要：不要基于任何旧图修图，不要复刻参考图的噪点、污渍分布、比例和空白。请重新设计一张更紧凑、更均衡、更适合填字的干净空壳。
重要：由于生成式 AI 容易写错中文，除本提示明确要求的区域标题和少量固定字段标签外，其他文本全部留空。尤其是技能区不要写任何技能名称。

整体风格：
- 牛皮档案纸、1920 年代调查档案、克苏鲁神话低调暗纹、旧书页质感。
- 低饱和、浅色、可读性优先。
- 细边框、圆角矩形区域、打字机字体标签、旧金线条、暗红印章点缀；纹理必须轻，不要脏。
- 不要画人物，不要画真实二维码，不要写任何具体调查员姓名。
- 视觉重心要均衡，不要头重脚轻；中上部要紧凑，下半部分要有足够高度。

字号定义：
- 小字：12px
- 中字：16px
- 大字：24px
- 特大字：32px

版面总体约束：
- 外边距约 36-48px。
- 所有区域使用浅色圆角矩形，圆角约 12-18px。
- 区域之间垂直间距保持 16-24px，不要大面积空白。
- 动态文字区域必须干净、浅色、低对比，便于后续叠加黑色中文。
- 固定标签用打字机风格中文字体或清晰宋体风格，不要手写体。
- 所有位置关系按百分比理解，不要因为实际输出尺寸略有偏差而互相遮挡。

必须包含以下固定结构与固定标签：

1. 标题行：
- 标题行位于整张图顶部 4%-13% 高度范围。
- 左上写小字：coc7th人物卡 by 白羽
- 下方在左侧 5%-66% 宽度范围内留出一条调查员姓名空行，后续会由脚本用特大字写入姓名；姓名空行不要太高。
- 标题行右侧增加一个显眼头像区域：竖向复古相片框，位置固定在整张图右上 69%-95% 宽度、3%-26% 高度范围内。
- 头像框不得进入左侧标题文字区和姓名空行，头像框与姓名空行之间至少保留 2% 画布宽度的间距。
- 头像相框要比普通信息框更显眼：暗红外框、旧金内框、轻微投影、旧照片角标；内部保持空白或极淡纸纹。
- 头像区域不要生成具体人物、脸、剪影、文字或图标。

2. 固有属性行，左右结构：
- 该行位于整张图 15%-31% 高度范围。
- 左侧区域标题：基本背景块
- 基本背景块不要松散，内部六行紧凑排列：出生年月日、职业、出生地、居住地、母语、目前国籍。
- 基本背景块每行左侧只写固定标签，右侧只留干净的浅色空白填写区域；不要画横线、虚线、短横、冒号、下划线、破折号或任何占位符。
- 基本背景块宽度约占该行 34%-38%，不要过大。
- 右侧区域标题：属性值块
- 属性值块宽度约占该行 58%-62%。
- 属性值块内有 8 个紧凑小方块，4列2行：力量 STR、体质 CON、体型 SIZ、敏捷 DEX、外貌 APP、智力 INT、意志 POW、教育 EDU。
- 每个属性小方块内只需要容纳两位数字，不要留巨大空白；数字空位约 42x34px 即可。
- 属性小方块要像精致 UI badge，不要像大输入框。

3. 衍生属性与技能行，左右结构：
- 该行位于整张图 32%-56% 高度范围。
- 左侧区域标题：基本数值块
- 基本数值块的 UI 排布必须和属性值块一致，使用紧凑小方块，不要用横线列表。
- 基本数值块内有 9 个小方块，建议 3列3行。
- 顺序必须是：生命值 HP、魔法值 MP、理智值 SAN、幸运 LUCK、移动力 MOV、伤害加值 DB、体格 BUILD、闪避 Dodge、体型 SIZ。
- 前四个 HP、MP、SAN、幸运放在最前面，视觉上可以略微强调。
- 每个小方块内保留数值空位，能容纳 `10/10`、`65/99`、`70` 这类短值。
- 右侧区域标题：技能值块
- 技能值块也使用小方块 UI，不要使用长表格横线。
- 技能值块内只画空白小方块网格，不要写任何技能名称，不要写“技能名称/技能值”，不要写“自定义”。
- 每个技能小方块分为左侧较宽的技能名空位和右侧较窄的数值空位，后续都由脚本填写。
- 技能方块尽量紧凑，建议 4列或5列网格，总数约 32-40 个空技能格。
- 技能区不要出现任何中文技能文字、英文技能文字、数字或伪文字。

4. 随身物品行：
- 一个横向浅色圆角矩形区域，标题：随身物品行
- 该区域要比参考图更高，约 180-220px。
- 内部留出多行干净空白，后续写道具、武器、装备。

5. 背景故事组合区：
- 背景故事主块和背景故事补充块要拼成一个整体的圆角六边形叙事区，不再是两个完全分离的矩形。
- 形状要求：整体看起来像上下相连的双段档案卡，中间边缘可以有折角/斜切/六边形转折；外轮廓是圆角六边形或拼接式圆角六边形。
- 上半段标题：背景故事行
- 下半段标题：背景故事（补充）
- 整个组合区位于随身物品行下方，占整张图约 63%-90% 高度范围，是整张卡最大的叙事区域。
- 上半段和下半段都保留大面积干净空白，后续由脚本写背景故事和补充内容。

6. 底部落款行，左右结构：
- 左侧不再单独做白羽档案馆，不再单独做背景故事（补充）矩形；左侧空间已经并入上面的背景故事组合区。
- 右侧区域保留为“白羽档案馆”信息区，顶部小字写：白羽档案馆。
- 右侧信息区位于底部右侧，占整张图约 69%-95% 宽度、90%-97% 高度范围；如果实际输出高度略小，也必须保证该区域内文字空位和二维码空位都完整可见。
- 右侧信息区内部必须左右分栏：左侧 55%-62% 宽度留给三行社媒账号文字空位，右侧 32%-38% 宽度留给正方形二维码空位。
- 社媒文字空位只画三条干净浅色文本承载区，不要写任何账号、平台名、伪文字或乱码。
- 二维码空位保持正方形，尺寸不要超过右侧信息区高度的 68%，不要挤占左侧社媒文字空位。
- 白羽档案馆标题、社媒文字空位、二维码空位之间必须有清晰间距，不能让二维码框把整个右下角占满。
- 底部左侧可留很小的装饰或空白，不要另外放独立社媒信息块。

严格限制：
- 只允许写上述固定标签，不要写任何动态内容。
- 不要写示例姓名、示例属性值、示例技能、示例故事。
- 技能区严禁出现任何技能名称，所有技能名称和数值都由脚本后续填写。
- 基本背景块右侧填写区严禁出现横线、短横、虚线、下划线或任何占位符，只能是空白浅色输入区域。
- 右下角白羽档案馆区域必须同时保留社媒文字空位和二维码空位，不能只剩二维码框。
- 不要在头像区域生成具体人物、脸、剪影或头像文字。
- 不要生成真实二维码。
- 不要使用旧图作为底图，不要模拟旧图的噪点，不要让纸张纹理过重。
- 所有空白区域必须足够干净，方便叠加黑色中文文字和数字。
- 框线和纹理不要太重，不能干扰阅读。
- 固定标签中文字必须准确，不要错字、乱码、伪文字。
- 版面要紧凑但不拥挤，避免上半部分巨大留白，避免下半部分过矮。
```

输出后放到：

tools/coc-card-renderer/assets/card_shell.png

## 2. 属性小图标套组

```text
生成一组克苏鲁调查员档案 UI 小图标，透明背景 PNG，风格统一，暗红、旧金、墨黑配色，线条清晰。图标包括：力量、体质、体型、敏捷、外貌、智力、意志、教育、生命值、理智值、魔法值、移动力。不要写文字。每个图标独立、居中、适合缩放到 64x64。
```

## 3. 角标与分隔线装饰

```text
生成透明背景 PNG 装饰素材，用于克苏鲁风格人物卡 UI：四角金色档案角标、暗红印章边框、细长分隔线、旧纸标签条、神秘学圆形纹章。不要写文字。线条干净，适合叠加在浅色羊皮纸背景上。
```

## 4. “白羽档案馆”社媒区域装饰

```text
生成透明背景 PNG 小型装饰框，用于图片右下角社媒和二维码区域。风格为 1920 年代调查档案、旧金细框、暗红蜡封印章、小羽毛笔触。需要保留一个正方形二维码空位和旁边三行文字空位。不要生成真实二维码，不要写具体账号。
```

## 5. 大成功 / 大失败印章

```text
生成两个透明背景 PNG 印章素材，克苏鲁跑团风格，一个写“大成功”，一个写“大失败”。暗红色旧印章质感，略微倾斜，边缘磨损，适合盖在人物卡角落或骰点结果图上。中文字必须清晰准确。
```
